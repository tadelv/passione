# Impeccable Design Context — Passione

## Design Context

### Users
Espresso enthusiasts who own a Decent Espresso DE1 machine. They interact with the UI on a tablet (primary) or phone/desktop (secondary) while making coffee — often with wet hands, in a hurry, or mid-workflow. The job: control the machine confidently, monitor shots in real time, and dial in their espresso over time. They care deeply about the craft and expect the interface to respect that.

### Brand Personality
**Warm, inviting, confident.** Passione should feel like it was made by someone who loves both great software and great espresso. Not cold or clinical — it has warmth and personality — but it always knows what it's doing. The name means "passion" in Italian, and the interface should honor that: purposeful, considered, a pleasure to use.

### Aesthetic Direction
**"What would a coffee machine UI look like if Apple designed it?"**

- **Dark mode only** — immersive, distraction-free, appropriate for kitchen/café environments
- **Apple Home / Apple Watch** as the primary reference — clean dark surfaces, purposeful information density, confident typography, subtle depth through layering
- **Not a dashboard** — avoid generic SaaS / admin panel / Tailwind-default aesthetics. Every screen should feel purpose-built for espresso
- **Themed** — 5 built-in color presets (default navy, midnight, espresso, nordic, ocean) with full user customization via color picker
- **Touch-first** — minimum 44px touch targets, two-step confirmations for operations, drag-to-adjust inputs

### Design Principles

1. **Purposeful density** — Show the right information at the right time. During a shot, the graph and live metrics dominate. At idle, quick-start actions and last-shot summary take focus. Never show everything at once; never hide what matters now.

2. **Confidence through clarity** — Large, readable values. Unambiguous states. The user should never wonder "is it brewing?" or "did that work?" Machine state drives the UI, not the other way around.

3. **Warmth without whimsy** — Soft transitions, comfortable spacing, inviting colors. But no bouncy animations, no mascots, no gamification. Respect the user's expertise and time.

4. **Touch-native feel** — Every interaction should feel physical and responsive. Press-and-hold, drag-to-adjust, two-step confirms. Designed for fingers first, mouse second, keyboard as a power-user bonus.

5. **Invisible complexity** — Profiles, recipes, shot history, settings — there's a lot here. Layer it so casual use is effortless and advanced use is discoverable. Progressive disclosure, not feature walls.

### Existing Design System

**Colors**: Dark navy base (`#1a1a2e`), blue primary (`#4e85f4`), red-pink accent (`#e94560`), chart colors mapped to espresso concepts (pressure=teal, flow=blue, temperature=red, weight=brown)

**Typography**: System font stack (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto`), fluid sizing with `clamp()`, 4-tier responsive scale (mobile/tablet/desktop/large)

**Spacing**: 8/16/24px scale with responsive adjustments. 12px/16px border radius. Status bar and bottom bar at 70px (48px mobile).

**Interactions**: ActionButtons with two-step confirm (tap → "Tap to start" → 2s timeout). ValueInput with drag, hold-repeat, and full keyboard support. Preset pills with tap/double-tap/long-press.

**Accessibility target**: WCAG AAA where practical — enhanced contrast ratios, comprehensive ARIA labeling, keyboard navigation, large touch targets (44–56px minimum).
